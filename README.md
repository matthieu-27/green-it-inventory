# Cas d'Étude : "Green-IT Inventory"

## Présentation du projet

La PME **"EcoLogic Solutions"**, spécialisée dans le conseil en développement durable, souhaite lancer un nouvel outil interne commercialisable : **"Green-IT Inventory"**.

Il s'agit d'une **plateforme SaaS** permettant aux entreprises de :
- Recenser leur parc informatique.
- Évaluer l'empreinte carbone de chaque matériel (serveurs, PC, smartphones).
- Générer un **rapport de durabilité réglementaire**.

## Contexte et Contraintes

- **Délai** : L'outil doit être prêt pour le **prochain salon de la Tech Verte dans 4 mois**.
- **Budget** : Limité à **60 000 €** pour la phase de lancement (MVP).
- **Qualité** : Haute précision attendue sur les calculs (normes ISO), mais interface simplifiée pour cette première version.
- **Concurrents** : Des grands cabinets d'audit ont déjà des outils complexes et très chers. L'atout ici est la **simplicité et le coût réduit**.

**Dilemme organisationnel** :
Le PDG hésite entre :
1. Une **organisation fonctionnelle** (Marc garde la main sur les développeurs au sein du service informatique).
2. Une **organisation par projet** (les développeurs sont détachés à 100% sous les ordres d'Alexandre).
